# 📡 Wi-Fi 接続監視システム（Windows版）

Wi-Fiの接続状況を常時監視し、切断が発生した際に自動で原因を診断・ログに記録するツールです。  
切断直前の電波強度・Ping結果・周辺APのスキャン結果などをもとに、原因の推定と対処法を表示します。  
約10分ごとに実際の通信速度（ダウンロード・アップロード）も自動計測・記録します。  
RSSI・SNR・アクティブ機器数・現在地情報も同時に記録し、時系列グラフで相関分析が可能です。

---

## 📁 ファイル構成

```
C:\WiFiMonitor\
├── wifi_monitor.py       ← 監視プログラム本体
├── log_viewer.html       ← ログ確認用ビューア（ブラウザで開く）
├── 起動.bat              ← 手動起動用バッチファイル（自分で作成が必要）
├── 完全削除.bat  ← システム完全削除用（任意の名前に変更可）
└── logs\                 ← ログ保存フォルダ（自動作成）
    └── wifi_log_YYYY-MM-DD.json
```

---

## ✅ 動作環境

- Windows 10 / 11
- Python 3.8 以上
- Windowsの位置情報サービスがON（現在地取得に必要）

---

## 🔧 セットアップ手順（初回のみ）

### 1. ファイルの配置

1. このリポジトリのファイルをダウンロードする
2. `C:\` ドライブを開き、`WiFiMonitor` という名前のフォルダを作成する
3. ダウンロードした以下のファイルを `C:\WiFiMonitor\` に移動する
   - `wifi_monitor.py`
   - `log_viewer.html`

---

### 2. Python のインストール

1. https://www.python.org/downloads/ を開く
2. 「Download Python 3.x.x」の黄色いボタンをクリックしてダウンロード
3. ダウンロードした `.exe` ファイルを実行する
4. **⚠️ 重要：** インストール画面の一番下にある「**Add python.exe to PATH**」に必ずチェックを入れる
5. 「Install Now」をクリックしてインストール完了まで待つ

---

### 3. speedtest-cli のインストール

通信速度の自動計測に必要なライブラリをインストールします。  
コマンドプロンプトを開いて以下を実行してください。

```
"%LocalAppData%\Programs\Python\Python314\python.exe" -m pip install speedtest-cli
```

> ⚠️ Python のバージョンが異なる場合（例：3.13など）は `Python314` の部分を対応するバージョンに変更してください。  
> バージョンの確認方法：コマンドプロンプトで `python --version` を実行する

---

### 4. Windowsの位置情報サービスをONにする

現在地の記録に使用します。

1. 「設定」→「プライバシー」→「位置情報」を開く
2. 「このデバイスの位置情報へのアクセス」をオンにする
3. 「アプリが位置情報にアクセスすることを許可する」をオンにする

---

### 5. 起動.bat の作成

1. `C:\WiFiMonitor\` フォルダを開く
2. フォルダ内の何もない場所で右クリック →「新規作成」→「テキストドキュメント」
3. ファイル名を `起動.bat` に変更する（`.txt` → `.bat` に変える）
   - 拡張子が表示されていない場合：エクスプローラーの「表示」→「ファイル名拡張子」にチェックを入れる
4. `起動.bat` を右クリック →「メモ帳で編集」
5. 以下をそのままコピーして貼り付ける

```bat
@echo off
"%LocalAppData%\Programs\Python\Python314\python.exe" "%~dp0wifi_monitor.py"
pause
```

> ⚠️ Python のバージョンが異なる場合は `Python314` の部分を変更してください。

6. メモ帳の「ファイル」→「名前を付けて保存」を開く
7. 文字コードを「**ANSI**」に変更して保存する

## 🖥️ 起動方法

### パターン A：手動で起動する場合

`C:\WiFiMonitor\起動.bat` をダブルクリックする。

以下のような画面が表示されれば正常に動作しています。

```
==============================================================
  Wi-Fi 接続監視システム  起動完了
  監視開始   : 2026-03-12 17:07:09
  ログ保存先 : C:\WiFiMonitor\logs
  監視間隔   : 5 秒
  Speedtest  : 約30分ごとに自動計測
  終了するには Ctrl + C を押してください
==============================================================
  [OK] speedtest-cli が使用可能です
  デフォルトゲートウェイ: 192.168.1.1
  📍 位置情報を取得中...
[17:07:09] ✅ 接続中 | BE001 | 電波:96% | Ch:100 | Rx:200Mbps
```

**終了するには** `Ctrl + C` を押す。  
ウィンドウの × ボタンを押すと確認ポップアップが表示されます。

### パターン B：PC起動時に自動で監視を開始する場合

Windowsのタスクスケジューラに登録することで、PCにログインするたびに自動で監視が始まります。

#### 手順

1. スタートメニューで「**タスクスケジューラ**」と検索して開く
2. 右側の「**タスクの作成**」をクリック

**「全般」タブ**
- 名前：`WiFi監視`
- 「ユーザーがログオンしているときのみ実行する」を選択
- 「最上位の特権で実行する」にチェックを入れる

**「トリガー」タブ**
- 「新規」をクリック
- タスクの開始：「**ログオン時**」を選択
- 「OK」をクリック

**「操作」タブ**
- 「新規」をクリック
- プログラム/スクリプト欄に入力：
  ```
  cmd.exe
  ```
- 引数の追加欄に入力：
  ```
  /k "C:\WiFiMonitor\起動.bat"
  ```
- 開始（オプション）欄に入力：
  ```
  C:\WiFiMonitor
  ```
- 「OK」をクリック

**「条件」タブ**
- 「AC電源が使用されている場合のみタスクを開始する」の**チェックを外す**（バッテリーでも動かすため）

3. 「OK」で保存する
4. タスク一覧に `WiFi監視` が「準備完了」の状態で表示されれば登録完了
5. PCを再起動してログインすると自動で監視画面が起動する

```
  ▲ アップロード: 47.3 Mbps  （実測値）
  🏓 Ping       : 40.6 ms
  サーバー      : Tsukuba
  計測時刻      : 2026-03-12 17:01:18
--------------------------------------------------------------
  次の詳細更新まで 60 秒  |  次のSpeedtestまで約10分  |  終了: Ctrl+C
==============================================================
```

### 表示項目の説明

| 項目 | 内容 |
|------|------|
| アクティブ機器 | ARPテーブルベースの同一ネットワーク上のアクティブ機器数（参考値） |
| 現在地・緯度経度 | WindowsのLocation APIで取得した位置情報 |
| RSSI | 電波強度（dBm）。-50以上が優良、-70以下で不安定 |
| SNR（推定） | 信号対雑音比。40dB以上が優秀、25dB未満は不安定 |

> ℹ️ アクティブ機器数はルーター管理画面の数値とは異なります。自分のPCが通信した機器のみが対象となるため、ネットワーク全体の正確な台数ではなく増減の傾向を見るための参考値です。

> ℹ️ SNRはWindowsが標準で公開していないため、RSSI - 推定ノイズフロア（-95dBm）で計算した近似値です。

## 📊 ログの確認方法

ログは `C:\WiFiMonitor\logs\` に日付ごとのJSONファイルとして保存されます。  
（例：`wifi_log_2026-03-12.json`）

### ログビューアで確認する

1. `log_viewer.html` をブラウザ（Edge / Chrome）でダブルクリックして開く
2. エクスプローラーで `C:\WiFiMonitor\logs\` フォルダを開く
3. 確認したい日付の `.json` ファイルをブラウザの画面にドラッグ＆ドロップする
4. 時系列グラフとイベント一覧が表示される

### 時系列グラフ

| 色 | データ | デフォルト |
|----|--------|-----------|
| 🟢 緑 | 電波強度 (%) | ON |
| 🔵 青 | リンク速度Rx (Mbps) | ON |
| 🟠 橙 | DL実測 (Mbps) | ON |
| 🟣 紫 | UL実測 (Mbps) | ON |
| 🔴 赤 | Ping遅延 (ms) | ON |
| 🟡 黄 | アクティブ機器数 (台) | ON |
| ⚪ 白 | リンク速度Tx (Mbps) | OFF |

グラフ右上のボタンで各データの表示・非表示を切り替えられます。

### ログのイベント種別

| アイコン | 種別 | 内容 |
|---------|------|------|
| 📊 | ステータス | 60秒ごとの定期記録。Wi-Fi情報・RSSI・機器数・位置情報・Speedtest結果を含む |
| ❌ | 切断 | 切断検知時の診断レポート |
| 🟢 | 再接続 | 再接続検知時の記録 |
| 📶 | Speedtest | 速度計測結果 |
| 💤 | スリープ復帰 | スリープ・休止状態からの復帰を検知した記録 |

> ℹ️ ステータスエントリを展開すると、位置情報セクションに「🗺️ 地図を開く」リンクが表示されます。クリックするとGoogleマップが別タブで開きます。

## 🔍 切断時の動作

Wi-Fiが切断されると自動で以下の診断が実行され、結果が画面に表示されます。

```
[13:31:45] ❌ 切断を検出！ 診断を実行中...

  Ping → 192.168.1.1 ... ❌ タイムアウト  ロス:100%
  Ping → 8.8.8.8 ...    ❌ タイムアウト  ロス:100%
  Ping → 1.1.1.1 ...    ❌ タイムアウト  ロス:100%

==============================================================
  切断診断レポート  [13:31:52]
==============================================================
【考えられる原因】
  ・ゲートウェイ・外部サーバーともに到達不能

【推奨対処】
  → ルーターの再起動を試みてください
==============================================================
```

| 診断内容 | 検出できる原因 |
|---------|--------------|
| 電波強度の記録 | 電波が弱い・不安定 |
| ゲートウェイへのPing | ルーターの障害 |
| 外部サーバーへのPing（8.8.8.8 / 1.1.1.1） | ISP側の障害 |
| 周辺APのスキャン | チャンネル干渉 |
| Windowsイベントログ | ドライバ・OS側の問題 |

## 📶 Speedtest 自動計測

以下のタイミングで実際の通信速度を自動計測し、ログに記録します。

| タイミング | 内容 |
|-----------|------|
| 起動時 | 即時1回計測 |
| 定期計測 | 約30分ごとに自動計測 |
| 再接続後 | 切断から復帰するたびに1回計測 |

計測はバックグラウンドスレッドで実行されるため、監視ループには影響しません。

> ⚠️ 会社のネットワークでは外部へのSpeedtest通信がセキュリティポリシーで制限されている場合があります。情報システム部に確認の上ご利用ください。

## 📍 位置情報

起動時・30分ごと・スリープ復帰時にWindowsの位置情報サービスから現在地を取得します。

| タイミング | 内容 |
|-----------|------|
| 起動時 | 即時取得 |
| 定期更新 | 30分ごとに再取得 |
| スリープ復帰検知時 | 即時再取得（事務所間移動に対応） |

住所はOpenStreetMap（nominatim）の逆ジオコーディングで日本語に変換されます。

> ℹ️ GPSチップがないPCでは、周辺Wi-FiのBSSIDや基地局情報をWindowsが参照して位置を推定します。IPジオロケーションより精度が高いですが、数十〜数百メートルの誤差が生じる場合があります。

---

## 💤 スリープ・休止状態への対応

スリープ中は監視が停止します。復帰後は自動的に以下の処理が実行されます。

1. スリープ復帰を検知してコマンドプロンプトに通知を表示
2. 位置情報を即時更新（事務所移動後に正確な現在地を取得）
3. `WAKE` イベントをログに記録（スリープ時間も記録）

> ⚠️ 長時間のスリープが問題になる場合は、電源設定でスリープをなしに設定することで監視を継続できます。  
> 「設定」→「システム」→「電源とスリープ」→「スリープ」→「なし」

## 🗑️ 完全削除

`完全削除.bat`（ファイル名は任意に変更可）を実行すると以下を自動削除します。

| 削除対象 | 内容 |
|---------|------|
| `C:\WiFiMonitor\` | プログラム・ログ・ビューアすべて |
| タスクスケジューラ「WiFi監視」 | 自動起動タスク |
| speedtest-cli | Pythonパッケージ |

> ℹ️ バッチファイル自体は `C:\WiFiMonitor\` ごと削除されるため、実行後に自分自身だけ残ります。手動で削除してください。

> ℹ️ バッチファイルの名前はどんな名前に変更しても正常に動作します。

## ❓ トラブルシューティング

### 起動.bat をダブルクリックしてもすぐ閉じる

Python のパスが異なる可能性があります。コマンドプロンプトを開いて以下を実行し、表示されたパスで `起動.bat` の1行目を修正してください。

```
where python
```

### 「❌ 未接続」と表示され続ける

Wi-Fiに接続されているにもかかわらず未接続と表示される場合、日本語Windowsの表示の違いによる問題の可能性があります。コマンドプロンプトで以下を実行して「状態」の項目の文字列を確認してください。

```
netsh wlan show interfaces
```

`wifi_monitor.py` の該当行にある以下のコードに確認した文字列を追加してください。

```python
is_connected = state_val in ("connected", "接続済み", "接続", "接続されました", "ここに追加")
```

### 監視が数分間止まって見える

コマンドプロンプトのウィンドウをクリックすると「選択モード」に入り、プログラムの出力が一時停止します。タイトルバーに「選択」と表示されている場合はキーボードの何かのキーを押すと再開します。

**恒久的な対策：**  
コマンドプロンプトのタイトルバーを右クリック →「プロパティ」→「オプション」タブ →「簡易編集モード」のチェックを外す

### Speedtest が動かない

speedtest-cli がインストールされていない可能性があります。以下を実行してください。

```
"%LocalAppData%\Programs\Python\Python314\python.exe" -m pip install speedtest-cli
```

### Speedtest エラー: HTTP Error 403

会社のネットワークで特定のSpeedtestサーバーへのアクセスがブロックされている場合があります。次回の自動計測（約10分後）で別のサーバーが選ばれて解消されることが多いです。繰り返し発生する場合は情報システム部に確認してください。

### 現在地が取得できない・UNKNOWN と表示される

Windowsの位置情報サービスがOFFになっている可能性があります。「設定」→「プライバシー」→「位置情報」を確認してください。

### RSSI が正の値（例：100）になっている

RSSIは通常マイナス値（例：-52 dBm）です。正の値が表示される場合は取得に失敗しています。signal% からの逆算が正常に動作していないため、`wifi_monitor.py` を最新版に更新してください。
